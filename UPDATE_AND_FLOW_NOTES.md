# Resonara Bedienfluss und Updates

Diese Version reduziert automatische Sprünge in der Oberfläche. Bereiche werden nur noch sanft sichtbar gemacht, wenn der nächste sinnvolle Bereich außerhalb des Sichtfelds liegt. Die Resonanzwege werden zuerst ausgewählt und erst mit „Diesen Weg starten“ gestartet.

Für neue App-Versionen nutzt Resonara den Service-Worker-Updatefluss: Sobald GitHub Pages eine neue Version ausliefert und der Browser sie erkennt, erscheint ein kundenfreundlicher Hinweis „Neue Resonara-Version verfügbar“. Mit „Neue Version installieren“ wird die neue Version aktiviert und die App lädt einmal neu. Lokale Einträge bleiben erhalten, weil sie im Browser-Speicher liegen.
